Reports @FXML fields in controller classes whose declared type does not match the element type in the corresponding FXML file.

For example, if an FXML file declares <TextField fx:id="nameField"/> but the controller has @FXML private Label nameField;, this is a type mismatch that will cause a ClassCastException at runtime.

The field type must be the same as the FXML element type, or one of its supertypes.

Use the quick fix to change the field type to match the FXML declaration.
